Master HTML

You must first understand the fundamentals of HTML before diving into the storyline.

1. The Basic Boilerplate

Every HTML document starts with a fundamental structure. Here is the classic blank canvas.

<!DOCTYPE html>
<html lang="en">
<head>
    <title>My First Page</title>
</head>
<body>
    <h1>Hello, World!</h1>
</body>
</html>

<!DOCTYPE html>

Declares the document type and tells the browser to render the page using HTML5 standards.

<head>

Contains meta-information about the webpage, like the title, character set, and links to stylesheets.

<body>

The container for all the visible content of your webpage, such as headings, paragraphs, and images.

2. Elements & Tags

HTML uses tags to structure text. Most elements have an opening tag and a closing tag.

<h1>Main Heading</h1>
<h2>Subheading</h2>
<p>This is a standard paragraph of text.</p>

<!-- Self-closing tag -->
<hr>
<br>

Headings (h1 - h6)

Defines headings on your page. <h1> is the most important (largest), down to <h6>.

Paragraphs (<p>)

Used for blocks of text. Browsers automatically add empty space (margins) before and after paragraphs.

Self-Closing Tags

Some tags like <br> (line break) or <hr> (horizontal rule) don't wrap content and don't need a closing tag.

3. Attributes

Attributes provide additional information about HTML elements and are always specified in the start tag.

<p class="highlight">This text has a class attribute.</p>
<div id="main-section">This div has a unique ID.</div>
<p title="Tooltip text!">Hover over me.</p>

class

Assigns one or more class names to an element. Used heavily by CSS to style multiple elements at once.

id

Assigns a unique identifier to an element. No two elements on a page should have the same ID.

title

Provides extra information about an element, typically displayed as a tooltip when hovered.

4. Links & Images

Connect your pages to the web and embed rich media like images.

<!-- Anchor (Link) -->
<a href="https://example.com" target="_blank">Visit Example</a>

<!-- Image -->
<img src="hero.jpg" alt="A brave knight" width="300">

Anchor Tag (<a>)

Defines a hyperlink. The href attribute specifies the destination URL.

Image Tag (<img>)

Embeds an image. Requires the src (source) attribute to locate the image file.

alt Attribute

Provides alternative text for an image if it fails to load, crucial for accessibility and screen readers.

5. Lists

Organize related items using ordered or unordered lists.

<h3>Unordered List</h3>
<ul>
    <li>Swords</li>
    <li>Shields</li>
</ul>

<h3>Ordered List</h3>
<ol>
    <li>First quest</li>
    <li>Second quest</li>
</ol>

<ul>

Creates an unordered list, typically displayed with bullet points.

<ol>

Creates an ordered list, typically displayed with numbers or letters.

<li>

Stands for "List Item". Used inside both <ul> and <ol> elements.

6. Tables

Display data in a structured grid of rows and columns.

<table border="1">
    <tr>
        <th>Class</th>
        <th>HP</th>
    </tr>
    <tr>
        <td>Mage</td>
        <td>80</td>
    </tr>
</table>

<table>

The container element that wraps all the data and structure for your table.

<tr> & <th>

<tr> creates a table row. <th> creates a table header cell (usually bold and centered).

<td>

Table Data. These are the standard cells that hold your actual data inside a row.

7. Forms

Collect user input using text fields, radio buttons, and submit buttons.

<form action="/submit" method="POST">
    <label for="username">Username:</label>
    <input type="text" id="username" name="username">

    <label for="password">Password:</label>
    <input type="password" id="password" name="password">

    <button type="submit">Login</button>
</form>

<form>

The shell for collecting input. The action attribute dictates where data goes, and method determines how it gets sent.

<input>

A highly versatile tag. Depending on its type attribute, it can be a text box, checkbox, radio button, or date picker.

<label>

Provides a text label for an input. The for attribute should match the input's id for accessibility.
