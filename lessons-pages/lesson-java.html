Master Java

You must first understand the fundamentals of Java before diving into the storyline.

1. The Basic Boilerplate

Every Java program is built inside a class and requires a main method to run. Here is the classic "Hello World".

public class Main {
    public static void main(String[] args) {
        System.out.println("Hello, World!");
    }
}

public class Main

Every line of code that runs in Java must be inside a class. The class name should generally match the filename.

main(String[] args)

The main method is the entry point of your program. Execution always starts from here.

System.out.println()

The standard command to print text to the console. It prints the text and moves the cursor to a new line.

2. Variables & Data Types

Java is strongly typed, meaning you must declare the type of a variable before using it to store data.

int level = 5;               // Integer (whole numbers)
double health = 99.5;        // Double (floating-point numbers)
char rank = 'A';             // Character (single letter)
boolean isGameOver = false;  // Boolean (true or false)

Numeric Types

Use int for standard whole numbers and double or float when you need decimal precision.

char

Stores a single character. It is always surrounded by single quotes (e.g., 'Z').

boolean

Represents a boolean truth value. It can only evaluate to true or false.

3. Control Flow

Direct the logic of your program using conditional statements and loops.

int score = 85;

// Conditional Statement
if (score >= 90) {
    System.out.println("Excellent!");
} else if (score >= 75) {
    System.out.println("Passed!");
} else {
    System.out.println("Keep trying!");
}

// Loop Statement
for (int i = 1; i <= 3; i++) {
    System.out.println("Spawning enemy " + i);
}

if / else if / else

Allows your program to make decisions and execute different blocks of code based on whether a condition is true or false.

for loop

Used to repeat a specific block of code a known number of times. Contains an initialization, a condition, and an increment.

while loop

Repeats a block of code as long as a specified condition remains true. Useful when the number of iterations is unknown.

4. Methods

Reusable blocks of code designed to perform a specific task. In Java, functions are called methods and belong to classes.

// Method Declaration
public static int addNumbers(int a, int b) {
    return a + b;
}

public static void main(String[] args) {
    // Method Call
    int result = addNumbers(10, 5);
    System.out.println("The sum is: " + result);
}

Return Type

Specifies what type of data the method sends back (e.g., int, String). Use void if it returns nothing.

Parameters

Variables passed into the method (like int a, int b) so it has the data it needs to operate.

5. Arrays & Strings

Store multiple items of the same data type sequentially, or handle text efficiently using the built-in String class.

// Array of integers
int[] nums = {10, 20, 30, 40, 50};
System.out.println("First number: " + nums[0]); // Outputs 10

// String manipulation
String text = "Java Programming";
System.out.println("Length: " + text.length());

Arrays

A fixed-size sequence of elements of the same type. Accessed using a zero-based index (e.g., nums[0]).

Strings

Unlike basic types, String is an object in Java. It includes helpful built-in methods like .length() and .toUpperCase().

6. Object References

Java doesn't use explicit pointers. Instead, variables holding objects actually hold a "reference" (a memory address) to that object.

Player p1 = new Player();
p1.hp = 100;

// p2 now points to the exact same object in memory as p1
Player p2 = p1; 

p2.hp = 50; 

// Outputs 50, because p1 and p2 modify the same object!
System.out.println("P1 HP: " + p1.hp);

Primitive vs Reference

Primitive types (like int) store actual values. Reference types (like Objects) store a memory address pointing to the data.

Pass by Value

Java is strictly pass-by-value, but when you pass an object to a method, you are passing the *value of the reference*.

Garbage Collection

When an object no longer has any references pointing to it, Java automatically deletes it from memory.

7. Classes & Objects

Java is Object-Oriented. Classes act as blueprints to group related variables (fields) and methods together into one custom data type. 

// Define a class
class Player {
    String name;
    int hp;
    int level;
}

public class Main {
    public static void main(String[] args) {
        // Create a new player object using 'new'
        Player hero = new Player();

        hero.name = "Knight";
        hero.hp = 100;
        hero.level = 5;

        System.out.println("Hero Name: " + hero.name);
        System.out.println("Hero HP: " + hero.hp);
    }
}

Class

A blueprint for creating objects. It defines the state (variables) and behavior (methods) the object will have.

The 'new' Keyword

Used to dynamically allocate memory and instantiate a brand-new object from a class definition.

Dot Operator

Use the dot operator to access fields and methods belonging to an object, like hero.hp.
